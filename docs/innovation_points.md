# 项目创新点与加分项说明

## 1. 统一 Web 展示系统

本项目不是只提交脚本或单一数据库实验，而是构建了统一 Web 展示系统。系统将总览指标、统计图表、学者画像、Neo4j 知识图谱和向量检索集中在同一个前端界面中，便于课程验收时进行完整演示。

## 2. 面向科研数据的实体标准化

项目没有简单地把原始字段直接入库，而是针对 Web of Science 科研数据设计了实体清洗规则。尤其是学院实体提取部分，系统从地址字段中识别广东工业大学相关学院英文简称，并通过映射表统一为中文学院名称。对于无法确定的学院，系统保留原始短语或标记为“未知学院”，避免错误归类。

## 3. 多视角 Neo4j 图谱分析

知识图谱模块不只展示一张总图，而是拆分为学者合作网络、关键词与作者关联网络、学院合作关系图、最短合作路径和代表性子图五类视图。每类视图对应不同的 Cypher 查询和分析目标，使图数据库的关系分析能力更清晰。

## 4. 显式关系与语义相似的互补分析

向量检索本身是常规语义检索方法，本项目的加分点不在于“把文本转成向量”这一基础操作，而在于将向量语义相似与 Neo4j 显式合作关系放在同一个科研分析框架中。Neo4j 用于回答“谁和谁已经合作、通过什么路径关联”，向量检索用于回答“哪些论文或学者研究内容相似”。二者结合后，可以用于发现研究方向相近但尚未直接合作的潜在合作对象。

## 5. 可部署的课程展示形态

项目已补充 Render 公网部署配置，包括 `render.yaml`、`runtime.txt` 和 `.gitignore`。部署后可通过公网地址访问系统，便于教师远程查看。Render 部署时会自动安装依赖并构建向量索引，无需手动复制本地运行环境。
