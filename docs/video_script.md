# 5 分钟讲解视频稿

## 0:00-0:30 项目介绍

大家好，我的课程项目是“基于 NoSQL 的科研数据分析系统”。项目数据来源于教师提供的 Web of Science 学者科研数据，共包含 5000 条论文记录。系统围绕广东工业大学科研论文数据，完成了数据清洗、MongoDB 文档存储、Neo4j 学术知识图谱、向量语义检索和 Web 可视化展示。

## 0:30-1:10 数据清洗与结构化处理

首先是数据预处理。原始数据是 Web of Science 纯文本格式，每篇论文包含作者、标题、摘要、关键词、机构、期刊、年份、被引次数等字段，并以 ER 作为记录结束标记。系统通过 `scripts/prepare_data.py` 解析字段码，合并多行字段，并将作者、关键词、机构、参考文献等内容整理为结构化数组。对于学院字段，系统从地址字段中识别英文学院简称，并通过映射规则转换为中文学院名称。

## 1:10-1:50 MongoDB 存储与统计分析

在数据库存储方面，本项目选择 MongoDB 作为主存储数据库。原因是论文元数据属于半结构化数据，不同论文字段可能不完全一致，并且作者、关键词、机构、参考文献等字段天然适合用数组保存。系统将每篇论文保存为一个文档，并基于年份、学院、关键词、期刊等字段建立索引。前端总览页展示了论文总数、学者数量、关键词数量、期刊来源数量，以及年度发文趋势和学院发文量。

## 1:50-2:30 统计图表与学者画像

统计图表页面展示了高频关键词、主要期刊来源、WOS 学科类别和合作机构关系。通过这些图表可以看到，deep learning、optimization、feature extraction 等是数据集中的高频研究主题，电子工程、材料科学、人工智能和自动化控制是主要学科方向。学者画像页面支持输入作者姓名，例如 Liu, Bo，系统会返回该作者的论文数量、被引次数、学术年龄、合作作者、研究关键词和代表论文。

## 2:30-3:30 Neo4j 知识图谱

图谱部分使用 Neo4j 建模思想，将科研数据抽象为 Author、Paper、Institution、School、Keyword 和 Journal 六类节点，并构建 WROTE、HAS_KEYWORD、BELONGS_TO、AFFILIATED_WITH、PUBLISHED_IN 和 COAUTHORED 等关系。网站“知识图谱”页面提供五类展示：学者合作网络、关键词与作者关联网络、学院合作关系图、最短合作路径和代表性子图。通过这些视图，可以分析学者之间的合作强度、研究热点与作者的关系、跨学院合作情况，以及两位学者之间是否存在间接合作路径。

## 3:30-4:20 向量检索与相似推荐

向量检索模块用于补充图数据库的显式关系分析。系统将论文标题、摘要、作者关键词和 Keywords Plus 拼接为 `vector_text`，再使用 TF-IDF 生成文本向量。若环境安装 FAISS，则使用 `IndexFlatIP` 构建向量索引；若未安装，则使用 scikit-learn 余弦相似度作为兜底方案。网站“向量检索”页面支持输入研究主题检索相似论文，也支持输入作者姓名发现研究方向相近的学者。这样可以发现“主题相似但不一定已经合作”的潜在合作对象。

## 4:20-4:45 Web 系统与部署

本项目实现了统一 Web 展示系统，包含总览、统计图表、学者画像、知识图谱和向量检索五个页面。系统本地可通过 `uvicorn backend.main:app --host 127.0.0.1 --port 8000` 启动。为了支持公网访问，项目增加了 Render 部署配置文件 `render.yaml`，可通过 GitHub 连接 Render 生成公网地址。

## 4:45-5:00 总结与创新点

本项目的特点是将文档型数据库、图数据库、向量检索和前端可视化组合到同一个科研数据分析场景中。MongoDB 解决半结构化论文元数据存储问题，Neo4j 解决显式合作关系和路径分析问题，向量检索解决隐含语义相似问题，Web 前端则把多种分析结果统一展示。后续可以进一步接入更强的文本嵌入模型和作者消歧算法，提高推荐和图谱分析效果。
