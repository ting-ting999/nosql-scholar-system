# 5 分钟项目讲解稿（最终录制版）

## 0:00-0:25 项目介绍

【画面：打开公网首页，停留在“总览”页面】

大家好，我的课程项目是“基于 NoSQL 的科研数据分析系统”。项目使用教师提供的 Web of Science 科研数据，共包含 5000 条论文记录，主要围绕广东工业大学相关学者和科研成果展开。系统完成了数据清洗、MongoDB 文档存储设计、Neo4j 学术知识图谱、向量语义检索和统一 Web 可视化展示。

## 0:25-1:05 数据处理与 MongoDB

【画面：展示报告中的数据处理流程图或 prepare_data.py 代码截图】

原始数据采用 Web of Science 纯文本格式，每篇论文包含作者、标题、摘要、关键词、机构、期刊、年份和被引次数等字段，并以 ER 作为记录结束标记。系统通过 prepare_data.py 识别字段码、合并多行内容，并把作者、关键词、机构和参考文献整理为数组。

对于学院名称，系统从作者地址中识别英文简称，并通过映射表统一转换为中文学院名称。清洗后生成 papers.json、summary.json、Neo4j 导入 CSV 和向量检索数据。MongoDB 以 paper_id 作为唯一键保存论文文档，并对年份、学院、作者、关键词以及全文字段建立索引。

## 1:05-1:45 总览与统计图表

【画面：依次展示总览页和“统计图表”页面】

总览页面展示论文总数、学者数量、关键词数量和期刊来源数量，并通过年度发文与被引趋势、学院发文量反映科研产出变化。统计图表页面进一步展示关键词热点、主要期刊、WOS 学科类别和合作机构关系。

从结果可以看到，deep learning、optimization、feature extraction 等是主要研究主题，电子工程、材料科学、人工智能和自动化控制是数据集中的核心学科方向。

## 1:45-2:20 学者画像

【画面：进入“学者画像”，输入 Hao, Zhifeng，按回车】

学者画像页面支持通过按钮或回车查询作者。这里输入 Hao, Zhifeng，系统会返回该学者的论文数量、被引次数、首篇年份、学术年龄、研究关键词、合作作者和代表论文。该功能把分散在多篇论文中的作者信息聚合为可阅读的科研画像，也便于观察学者的主要研究方向和合作关系。

## 2:20-3:15 Neo4j 知识图谱

【画面：进入“知识图谱”，依次点击五个图谱标签】

图谱模块按照 Neo4j 建模方法设计了 Author、Paper、Institution、School、Keyword 和 Journal 六类节点，并构建 WROTE、HAS_KEYWORD、BELONGS_TO、AFFILIATED_WITH、PUBLISHED_IN 和 COAUTHORED 等关系。项目生成 Neo4j 导入 CSV，并通过 neo4j_import.cypher 完成节点和关系导入。

网站提供五种图谱视图，分别是学者合作网络、关键词与作者关联网络、学院合作关系图、最短合作路径和高被引论文代表性子图。右侧同时展示对应的 Cypher 查询。通过这些视图可以分析学者合作强度、热点方向下的核心作者、跨学院合作情况以及学者之间的间接合作路径。

## 3:15-4:10 向量检索与相似推荐

【画面：进入“向量检索”，等待默认结果出现；再输入 graph neural network，按回车】

向量检索模块将论文标题、摘要、作者关键词和 Keywords Plus 拼接为 vector_text，再使用 TF-IDF 生成并归一化文本向量。输入 graph neural network 后，系统会返回语义相似度最高的论文，并展示标题、年份、期刊、关键词和相似度分数。

右侧的相似学者功能将一名作者的多篇论文组合为研究画像，再计算作者之间的研究方向相似度。本地环境安装 FAISS 时可以使用 IndexFlatIP 检索；公网 Render 实例受 512MB 内存限制，因此使用约 2.2MB 的稀疏向量矩阵和 scikit-learn 余弦相似度，所以页面会显示 engine: sklearn。两种模式使用相同的文本表示和结果接口。

## 4:10-4:40 Web 系统与公网部署

【画面：回到首页，并在地址栏展示公网地址】

本项目将总览、统计图表、学者画像、知识图谱和向量检索集中在同一个 Web 系统中。后端采用 FastAPI，前端使用 HTML、CSS、JavaScript 和 ECharts。项目代码已上传至 GitHub，并部署到 Render，公网地址为 https://nosql-scholar-system.onrender.com。

为适应免费实例的内存限制，系统采用稀疏矩阵、轻量元数据、按需加载和单进程运行；ECharts 和图标文件也改为本地静态资源，减少外部网络依赖。

## 4:40-5:00 总结与创新点

【画面：快速切换总览、知识图谱和向量检索页面】

本项目的主要特点不是简单叠加多种技术，而是针对科研数据形成统一分析流程。MongoDB解决半结构化元数据存储问题，Neo4j分析已有合作关系和多跳路径，向量检索发现隐含的主题相似性；实体标准化、五类图谱视图和公网展示提高了系统的可解释性与完整性。以上就是本项目的主要内容，谢谢老师观看。
